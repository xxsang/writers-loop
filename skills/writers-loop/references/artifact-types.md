# Artifact Types

Use this reference as the routing index for artifact-specific planning and
critique guidance. Load only the category reference that matches the current
task.

If the artifact type is not listed here, use the universal rubric in
`references/critique-rubrics.md` and ask if it is sufficient. Do not invent
category rules that are not in the loaded reference.

## Routing

| User task | Load |
| --- | --- |
| Coding plans, implementation plans, technical proposals, technical documentation, design docs, product specs | `references/technical-writing.md` |
| Reports, memos, executive updates, proposals, business recommendations, academic papers, whitepapers, constrained corporate/legal/regulatory/brand writing | `references/business-writing.md` |
| Fiction chapters, narrative outlines, essays, scripts, speeches, poetry, memoir or personal narrative | `references/fiction-narrative.md` |
| Style learning, style distillation, learned style packs | `references/style-distillation.md` |
| Translation with fidelity, voice, rhythm, formatting, terminology, or cultural effect concerns | `references/translation.md` |

## Shared Fallback

Use `references/critique-rubrics.md` for the Universal Rubric and shared
revision-proposal format when:

- the artifact does not fit a category above
- the category reference does not cover the user's specific artifact
- the task is a hybrid and no single category controls

When using the fallback, say which artifact-specific rules are missing and ask
whether the universal rubric is sufficient.

## Hybrid Routing

When a request spans categories, choose references by dominant audience and
primary deliverable:

- Executive decision with technical backing: load `references/business-writing.md`
  first, then `references/technical-writing.md` for the technical appendix.
- Technical plan for non-technical leadership: lead with recommendation,
  evidence gaps, and decision request; keep file map, interfaces, and tests in a
  technical appendix.
- Narrative or speech with business facts: load `references/fiction-narrative.md`
  for voice and structure, then use `references/business-writing.md` to check
  claims, evidence, and audience action.
- If no category controls, use `references/critique-rubrics.md` and ask whether
  the Universal Rubric is sufficient.

## Preference Scope

Keep learned preferences scoped to the artifact type and stage. Do not reuse
fiction preferences in technical plans, business-report preferences in poetry,
or external brand/legal constraints as personal preferences unless repeated user
decisions justify that scope.
